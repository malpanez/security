**Section 2: Install** (tags: [usbguard, enforce, install])
3. EPEL install for RHEL 8: `ansible.builtin.dnf` with `name: https://dl.fedoraproject.org/pub/epel/epel-release-latest-{{ ansible_distribution_major_version }}.noarch.rpm` state: present — when: `ansible_os_family == 'RedHat'` AND `_usbguard_needs_epel | bool`

5. Install usbguard (RedHat): `ansible.builtin.dnf` name: `{{ _usbguard_package }}` state: present — when: `ansible_os_family == 'RedHat'`
6. Install usbguard (Suse): `community.general.zypper` name: `{{ _usbguard_package }}` state: present — when: `ansible_os_family == 'Suse'`

**Section 3: Configure** (tags: [usbguard, enforce, configure])
7. `ansible.builtin.file` path: `{{ _usbguard_rules_dir }}` state: directory owner: root group: root mode: "0750"
8. `ansible.builtin.stat` path: `{{ _usbguard_initial_rules }}` register: `_usbguard_rules_existing`
9. `ansible.builtin.command` cmd: `usbguard generate-policy` register: `_usbguard_generated_policy` changed_when: false — when: `usbguard_allow_existing_devices | bool` AND `not _usbguard_rules_existing.stat.exists`
10. `ansible.builtin.copy` content: `"{{ _usbguard_generated_policy.stdout }}\n"` dest: `{{ _usbguard_initial_rules }}` owner: root group: root mode: "0640" — when: `usbguard_allow_existing_devices | bool` AND `not _usbguard_rules_existing.stat.exists` AND `_usbguard_generated_policy.stdout is defined` AND `_usbguard_generated_policy.stdout | length > 0`
11. `ansible.builtin.copy` content: `"{{ usbguard_extra_rules | join('\n') }}\n"` dest: `{{ _usbguard_rules_dir }}/10-extra-rules.conf` owner: root group: root mode: "0640" notify: Restart usbguard — when: `usbguard_extra_rules | length > 0`
12. `ansible.builtin.template` src: `usbguard-daemon.conf.j2` dest: `{{ _usbguard_daemon_conf }}` owner: root group: root mode: "0600" backup: true notify: Restart usbguard

**Section 4: Service** (tags: [usbguard, enforce, service])
13. `ansible.builtin.service` name: `{{ _usbguard_service }}` state: started enabled: true

**Section 5: Verify** (tags: [usbguard, enforce, verify])
14. `ansible.builtin.command` cmd: `systemctl is-active {{ _usbguard_service }}` changed_when: false register: `_usbguard_verify_active`
15. `ansible.builtin.assert` that: `_usbguard_verify_active.stdout == 'active'` fail_msg: "usbguard service is not active after enforcement"
16. `ansible.builtin.debug` msg with completion report: service active, ImplicitPolicyTarget={{ usbguard_implicit_policy }}, rules_dir={{ _usbguard_rules_dir }}

  <action>
Create `templates/usbguard-daemon.conf.j2` with the following exact content structure:

Line 1: `# Managed by Ansible - do not edit`
Then key=value pairs, one per line, no section headers:

```
RuleFile={{ _usbguard_conf_dir }}/rules.conf
RuleFolder={{ _usbguard_rules_dir }}
ImplicitPolicyTarget={{ usbguard_implicit_policy }}
PresentDevicePolicy={{ usbguard_present_device_policy }}
PresentControllerPolicy={{ usbguard_present_controller_policy }}
InsertedDevicePolicy=apply-policy
RestoreControllerDeviceState={{ usbguard_restore_controller_state | ternary('true', 'false') }}
DeviceManagerBackend=uevent
IPCAllowedUsers=root
IPCAllowedGroups={{ _usbguard_ipc_group }}
IPCAccessControlFiles=/etc/usbguard/IPCAccessControl.d/
DeviceRulesWithPort=false
AuditBackend={{ usbguard_audit_backend }}
{% if usbguard_audit_backend == 'FileAudit' %}
AuditFilePath={{ usbguard_audit_file_path }}
{% endif %}
```

CRITICAL: Both `RuleFile` AND `RuleFolder` MUST be present. Without `RuleFolder`, files in rules.d/ are silently ignored.

The `_usbguard_ipc_group` variable resolves to `wheel` on RedHat/Suse or `sudo` on Debian (from OS vars).

`ternary('true', 'false')` converts Ansible boolean to usbguard's lowercase string format.
