  <action>
Create `roles/tls_hardening/tasks/review.yml` following the ntp_hardening review pattern (per STD-03, D-13, D-14, D-15, D-17, TLS-01):

1. **Get OpenSSL version** — `ansible.builtin.command: openssl version`, register `_tls_openssl_version`, `changed_when: false`, `failed_when: false`, tags: [tls_hardening, review]

2. **Read FIPS state** — `ansible.builtin.slurp` src `/proc/sys/crypto/fips_enabled`, register `_tls_fips_review`, `failed_when: false`, tags: [tls_hardening, review]. Use slurp not command (RESEARCH anti-pattern).

3. **Set FIPS fact** — `ansible.builtin.set_fact` `_tls_fips_active: "{{ (_tls_fips_review.content | default('MA==') | b64decode | trim) == '1' }}"`, tags: [tls_hardening, review]. Note: `MA==` is base64 for `0` (default when file absent).

4. **Get current crypto policy (RHEL 8/9 only)** — `ansible.builtin.command: update-crypto-policies --show`, register `_tls_rhel_policy`, `changed_when: false`, `failed_when: false`, when: `ansible_os_family == 'RedHat'` and `ansible_distribution_major_version | int >= 8` and `not _tls_in_container | bool` (per D-17 — skip in containers), tags: [tls_hardening, review]

5. **Slurp openssl.cnf (Debian/Suse/RHEL<8)** — `ansible.builtin.slurp` src `{{ tls_hardening_openssl_cnf }}`, register `_tls_openssl_cnf_review`, `failed_when: false`, when: `ansible_os_family in ['Debian', 'Suse'] or (ansible_os_family == 'RedHat' and ansible_distribution_major_version | int < 8)`, tags: [tls_hardening, review]

6. **Extract current MinProtocol** — `ansible.builtin.set_fact` `_tls_current_min_protocol` from slurped content using regex_search for `MinProtocol\s*=\s*(\S+)` with default `'NOT_SET'`, when: `_tls_openssl_cnf_review.content is defined`, tags: [tls_hardening, review]

7. **Extract current CipherString** — `ansible.builtin.set_fact` `_tls_current_cipher_string` from slurped content using regex_search for `CipherString\s*=\s*(.+)` with default `'NOT_SET'`, when: `_tls_openssl_cnf_review.content is defined`, tags: [tls_hardening, review]

8. **Report TLS review findings** — `ansible.builtin.debug` with structured msg list (per D-14):
   - "Host: {{ inventory_hostname }} ({{ ansible_distribution }} {{ ansible_distribution_version }})"
   - "OpenSSL version: {{ _tls_openssl_version.stdout | default('unknown') }}"
   - "FIPS active: {{ _tls_fips_active | default(false) }}"
   - "RHEL crypto policy: {{ _tls_rhel_policy.stdout | default('N/A (not RHEL 8/9 or container)') }}"
   - "Target RHEL policy: {{ tls_hardening_rhel_policy }}"
   - "RHEL policy change needed: {{ (_tls_rhel_policy.stdout | default('')) != tls_hardening_rhel_policy }}"
   - "openssl.cnf MinProtocol: {{ _tls_current_min_protocol | default('N/A') }}"
   - "openssl.cnf CipherString: {{ _tls_current_cipher_string | default('N/A') }}"
   - "Target MinProtocol: {{ tls_hardening_min_protocol }}"
   - "Target CipherString: {{ tls_hardening_openssl_cipher_string }}"
   tags: [tls_hardening, review]

All tasks MUST have `changed_when: false` or be `debug` tasks (per STD-03 — review is read-only). All modules use FQCN. All task names start uppercase.
  </action>
