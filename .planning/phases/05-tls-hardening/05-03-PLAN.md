<objective>
Create tasks/enforce.yml with two enforcement branches: (1) RHEL 8/9 crypto-policies via update-crypto-policies --set, and (2) Debian/SUSE/RHEL<8 openssl.cnf modification via slurp+blockinfile+lineinfile.

Purpose: Enforce minimum TLS version and cipher strength system-wide. RHEL 8/9 uses the native crypto-policies subsystem. All other platforms edit openssl.cnf with the full section linkage chain required by OpenSSL 3.x. Together, DEFAULT:NO-SHA1 (RHEL) and MinProtocol=TLSv1.2 (Debian/SUSE) satisfy TLS-04 (disable TLS 1.0 and 1.1 by default).
Output: enforce.yml that is idempotent, FIPS-aware, and triggers no service restarts.
</objective>

  <action>
Create `roles/tls_hardening/tasks/enforce.yml` with the following structure. CRITICAL: all tasks tagged with [tls_hardening, enforce] plus section-level tags (configure, verify).

**Section 1: FIPS detection (all platforms)**

1. **Enforce | Read FIPS state** — `ansible.builtin.slurp` src `/proc/sys/crypto/fips_enabled`, register `_tls_fips_raw`, `failed_when: false`, tags: [tls_hardening, enforce, configure]

2. **Enforce | Set FIPS fact** — `ansible.builtin.set_fact` `_tls_fips_active: "{{ (_tls_fips_raw.content | default('MA==') | b64decode | trim) == '1' }}"`, tags: [tls_hardening, enforce, configure]. Note: MA== is base64 for '0'.

3. **Enforce | Skip enforcement when FIPS active** — `ansible.builtin.debug` msg "FIPS mode is active - skipping TLS hardening enforcement (FIPS policy supersedes)", when: `_tls_fips_active | bool`, tags: [tls_hardening, enforce, configure] (per D-03)

**Section 2: RHEL 8/9 crypto-policies branch**

All tasks in this section have when conditions including: `ansible_os_family == 'RedHat'`, `ansible_distribution_major_version | int >= 8`, `not _tls_fips_active | bool`

4. **Enforce | Install crypto-policies-scripts (RHEL 8/9)** — `ansible.builtin.package` name `crypto-policies-scripts` state `present`, when: RHEL 8/9 + not FIPS, tags: [tls_hardening, enforce, install]

5. **Enforce | Read current crypto policy** — `ansible.builtin.command` cmd `update-crypto-policies --show`, register `_tls_current_policy`, `changed_when: false`, `failed_when: false`, when: RHEL 8/9 + not FIPS, tags: [tls_hardening, enforce, configure]

6. **Enforce | Set crypto policy** — `ansible.builtin.command` cmd `update-crypto-policies --set {{ tls_hardening_rhel_policy }}`, `changed_when: true`, when: RHEL 8/9 + not FIPS + `(_tls_current_policy.stdout | default('')) != tls_hardening_rhel_policy` (per D-04 — idempotent: only fires when policy differs). This enforces TLS-04 on RHEL: DEFAULT:NO-SHA1 disables TLS 1.0/1.1 by requiring TLS 1.2+. tags: [tls_hardening, enforce, configure]

**Section 3: openssl.cnf branch (Debian, SUSE, RHEL<8)**

7. **Enforce | Set openssl.cnf branch fact** — `ansible.builtin.set_fact` `_tls_use_openssl_cnf: "{{ ansible_os_family in ['Debian', 'Suse'] or (ansible_os_family == 'RedHat' and ansible_distribution_major_version | int < 8) }}"`, tags: [tls_hardening, enforce, configure]. CRITICAL: this set_fact MUST precede all steps 8-13. Every openssl.cnf-branch task (steps 8-13) MUST have BOTH `_tls_use_openssl_cnf | bool` AND `not _tls_fips_active | bool` in its when condition.

8. **Enforce | Slurp openssl.cnf** — `ansible.builtin.slurp` src `{{ tls_hardening_openssl_cnf }}`, register `_tls_cnf_content`, `failed_when: false`, when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool`, tags: [tls_hardening, enforce, configure]

9. **Enforce | Check if system_default_sect exists** — `ansible.builtin.set_fact` `_tls_sect_absent: "{{ _tls_cnf_content.content is not defined or '[system_default_sect]' not in (_tls_cnf_content.content | b64decode) }}"`, when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool`, tags: [tls_hardening, enforce, configure]

10. **Enforce | Add ssl_conf linkage to openssl_init** — `ansible.builtin.lineinfile` path `{{ tls_hardening_openssl_cnf }}`, regexp `^\s*ssl_conf\s*=`, line `ssl_conf = ssl_sect`, insertafter `\[openssl_init\]`, backup true. when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool`. tags: [tls_hardening, enforce, configure]. CRITICAL: this task runs whenever the openssl.cnf branch is active, REGARDLESS of `_tls_sect_absent`. The ssl_conf linkage under [openssl_init] must always be present for [system_default_sect] to take effect on OpenSSL 3.x (Pitfall 1 from RESEARCH). On systems where [system_default_sect] already exists but the chain link is absent, omitting this step would silently leave TLS-03 and TLS-04 unenforced.

11. **Enforce | Insert TLS policy block** — `ansible.builtin.blockinfile` path `{{ tls_hardening_openssl_cnf }}`, marker `# {mark} ANSIBLE MANAGED BLOCK tls_hardening`, insertafter `EOF`, block content (exactly):
    ```
    [ssl_sect]
    system_default = system_default_sect

    [system_default_sect]
    MinProtocol = {{ tls_hardening_min_protocol }}
    CipherString = {{ tls_hardening_openssl_cipher_string }}
    Options = ServerPreference
    ```
    backup true, when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool and _tls_sect_absent | bool`, tags: [tls_hardening, enforce, configure]. This enforces TLS-04 on Debian/SUSE: MinProtocol=TLSv1.2 disables TLS 1.0/1.1.

12. **Enforce | Ensure MinProtocol value** — `ansible.builtin.lineinfile` path `{{ tls_hardening_openssl_cnf }}`, regexp `^\s*MinProtocol\s*=`, line `MinProtocol = {{ tls_hardening_min_protocol }}`, insertafter `\[system_default_sect\]`, backup true, when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool and not _tls_sect_absent | bool`, tags: [tls_hardening, enforce, configure]

13. **Enforce | Ensure CipherString value** — `ansible.builtin.lineinfile` path `{{ tls_hardening_openssl_cnf }}`, regexp `^\s*CipherString\s*=`, line `CipherString = {{ tls_hardening_openssl_cipher_string }}`, insertafter `\[system_default_sect\]`, backup true, when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool and not _tls_sect_absent | bool`, tags: [tls_hardening, enforce, configure]

**Section 4: Verification read-back (STD-04 verify tag)**

14. **Enforce | Verify RHEL crypto policy applied** — `ansible.builtin.command` cmd `update-crypto-policies --show`, register `_tls_verify_policy`, `changed_when: false`, when: `ansible_os_family == 'RedHat' and ansible_distribution_major_version | int >= 8 and not _tls_fips_active | bool`, tags: [tls_hardening, enforce, verify]

15. **Enforce | Verify openssl.cnf MinProtocol applied** — `ansible.builtin.slurp` src `{{ tls_hardening_openssl_cnf }}`, register `_tls_verify_cnf`, when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool`, tags: [tls_hardening, enforce, verify]

16. **Enforce | Assert RHEL policy matches target** — `ansible.builtin.assert` that `_tls_verify_policy.stdout == tls_hardening_rhel_policy`, fail_msg "Crypto policy is {{ _tls_verify_policy.stdout | default('unknown') }}, expected {{ tls_hardening_rhel_policy }}", when: `ansible_os_family == 'RedHat' and ansible_distribution_major_version | int >= 8 and not _tls_fips_active | bool and _tls_verify_policy.rc | default(1) == 0`, tags: [tls_hardening, enforce, verify]

17. **Enforce | Assert openssl.cnf contains MinProtocol** — `ansible.builtin.assert` that `"'MinProtocol = ' ~ tls_hardening_min_protocol in (_tls_verify_cnf.content | b64decode)"`, fail_msg "MinProtocol not found in openssl.cnf after enforcement", when: `_tls_use_openssl_cnf | bool and not _tls_fips_active | bool and _tls_verify_cnf.content is defined`, tags: [tls_hardening, enforce, verify]

18. **Enforce | Report enforcement result** — `ansible.builtin.debug` msg with summary of what was applied (crypto-policies path or openssl.cnf path), tags: [tls_hardening, enforce, verify]
