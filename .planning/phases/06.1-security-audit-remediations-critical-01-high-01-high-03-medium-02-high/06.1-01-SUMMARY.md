# Phase 06.1 Plan 01: sssd_ad_integration Security Audit Remediations Summary

**MFA opt-in mechanism for corp LAN (AUDIT-CRIT-01), 7-day offline credential cache (AUDIT-HIGH-01), and strict group verification default (AUDIT-HIGH-03) added to sssd_ad_integration with backward-compatible defaults**

## Accomplishments

- Added `ssh_corp_require_mfa` variable controlling conditional `AuthenticationMethods` in the corp `Match Address` block — closes AUDIT-CRIT-01 mechanism, default `false` preserves backward compatibility
- Changed `sssd_offline_credentials_expiration` from `0` (never) to `7` (days) — closes AUDIT-HIGH-01: disabled AD accounts can no longer authenticate indefinitely when offline
- Changed `ssh_verify_groups_strict` from `false` to `true` — closes AUDIT-HIGH-03: unresolvable `AllowGroups` entries now fail the play instead of silently locking out all users
- Updated `argument_specs.yml` with full documentation and updated defaults for all 3 findings
- Updated `README.md` with new variable, changed defaults table, and upgrade notes for operators

## Decisions Made

- **AUDIT-CRIT-01 closed by opt-in:** `ssh_corp_require_mfa: false` default is a deliberate backward-compatibility choice. The finding is closed by providing the mechanism. Operators set `true` to enforce MFA on corp LAN.
- **`sssd_offline_credentials_expiration` 0->7:** 7 days balances offline access needs against the risk of indefinitely caching credentials for disabled AD accounts.
- **`ssh_verify_groups_strict` false->true:** Strict default prevents silent lockout — an unresolvable `AllowGroups` group in OpenSSH denies ALL users. Failing the play early is the safer behavior.
