Create `roles/user_audit/tasks/enforce.yml` with the following tasks. All enforce tasks that modify state MUST be gated with `when: not ansible_check_mode`. Use FQCN everywhere (STD-05). Tag all tasks with `[user_audit, enforce]`. Subdivide into configure and verify sections using comments (STD-04 — no install section needed since no packages to install).

**Section: configure**

1. **Lock accounts inactive beyond threshold** — use `ansible.builtin.user` (NOT `usermod -L`). Loop over intersection of lastlog output and human accounts to exclude system accounts:
```yaml
- name: Enforce | Lock accounts inactive beyond threshold
  ansible.builtin.user:
    name: "{{ item }}"
    password_lock: true
  loop: >-
    {{
      _ua_inactive_by_days.stdout_lines | default([])
      | intersect(_ua_human_account_list)
    }}
  when:
    - user_audit_lock_inactive | bool
    - item not in user_audit_skip_users
    - item != 'root'
    - not ansible_check_mode
  tags: [user_audit, enforce, configure]
```

The `intersect(_ua_human_account_list)` is the UID guard — prevents locking system accounts that appear in lastlog --before output. The `item not in user_audit_skip_users` and `item != 'root'` are the named and explicit root guards (triple guard: USR-04).

2. **Collect current maxdays for accounts without expiry** — two-step chage pre-check (step 1):
```yaml
- name: Enforce | Collect current maxdays for accounts without expiry
  ansible.builtin.command:
    cmd: chage -l {{ item }}
  loop: "{{ _ua_no_expiry | default([]) }}"
  register: _ua_chage_current
  changed_when: false
  failed_when: false
  when:
    - user_audit_set_password_expiry | bool
    - item not in user_audit_skip_users
    - item != 'root'
  tags: [user_audit, enforce, configure]
```

3. **Set expiry only where maxdays still -1 or 99999** — two-step chage (step 2). The regex matches both `-1` (Debian default) and `99999` (RHEL default):
```yaml
- name: Enforce | Set password expiry where maxdays is unlimited
  ansible.builtin.command:
    cmd: >-
      chage --maxdays {{ user_audit_max_password_age }}
            --mindays {{ user_audit_min_password_age }}
            --warndays {{ user_audit_warn_days }}
            {{ item.item }}
  loop: "{{ _ua_chage_current.results | default([]) }}"
  when:
    - item.stdout is defined
    - item.stdout is search('Maximum number of days between password change\s*:\s*(-1|99999)')
    - item.item not in user_audit_skip_users
    - item.item != 'root'
    - not ansible_check_mode
  changed_when: true
  failed_when: false
  tags: [user_audit, enforce, configure]
```

NOTE on the regex: Use single-quoted string for the `search()` filter to avoid YAML double-quote escaping issues with `\s` and `\(`. The task above uses single quotes around the regex.

4. **Fix service account shells** — optional, gated by `user_audit_fix_service_shells`:
```yaml
- name: Enforce | Set nologin shell on service accounts with interactive shells
  ansible.builtin.user:
    name: "{{ item }}"
    shell: "{{ user_audit_nologin_shell }}"
  loop: "{{ _ua_service_login_shells.stdout_lines | default([]) }}"
  when:
    - user_audit_fix_service_shells | bool
    - item not in user_audit_skip_users
    - item != 'root'
    - not ansible_check_mode
  tags: [user_audit, enforce, configure]
```

**Section: verify**

5. **Verify enforcement results** — re-collect and report:
```yaml
- name: Enforce | Verify password expiry enforcement
  ansible.builtin.command:
    cmd: chage -l {{ item }}
  loop: "{{ _ua_no_expiry | default([]) }}"
  register: _ua_enforce_verify
  changed_when: false
  failed_when: false
  when:
    - user_audit_set_password_expiry | bool
    - item not in user_audit_skip_users
    - item != 'root'
  tags: [user_audit, enforce, verify]

- name: Enforce | Report enforcement summary
  ansible.builtin.debug:
    msg:
      locked_candidates: >-
        {{
          _ua_inactive_by_days.stdout_lines | default([])
          | intersect(_ua_human_account_list)
          | difference(user_audit_skip_users)
        }}
      expiry_set_candidates: "{{ _ua_no_expiry | default([]) | difference(user_audit_skip_users) }}"
      service_shell_candidates: "{{ _ua_service_login_shells.stdout_lines | default([]) | difference(user_audit_skip_users) }}"
  tags: [user_audit, enforce, verify]
```

Key constraints:
- `ansible.builtin.user password_lock: true` is idempotent (USR-05) — module checks current state
- Two-step chage is idempotent (USR-05) — only fires when maxdays is -1 or 99999
- Triple guard on all enforce tasks (USR-04): intersect with human accounts OR skip_users + root check
- All `ansible.builtin.command` tasks that modify state have `when: not ansible_check_mode`
- `changed_when: true` on the chage set task (it genuinely changes state)
- `changed_when: false` on the chage collect and verify tasks (read-only)
