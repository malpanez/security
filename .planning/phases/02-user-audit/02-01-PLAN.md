8. Write `meta/argument_specs.yml` documenting every defaults variable. Use ntp_hardening argument_specs as structural reference. Variables to document:
   - user_audit_enabled (bool, default false) — gate variable, set to true with security_mode=enforce to apply enforcement
   - user_audit_inactive_days (int, default 90) — days since last login before an account is considered inactive for locking
   - user_audit_max_password_age (int, default 90) — maximum days between password changes (chage --maxdays), PCI-DSS 8.3.6
   - user_audit_min_password_age (int, default 1) — minimum days between password changes (chage --mindays)
   - user_audit_warn_days (int, default 14) — days before password expiry to warn the user (chage --warndays)
   - user_audit_lock_inactive (bool, default true) — when true, enforce locks accounts inactive beyond threshold
   - user_audit_set_password_expiry (bool, default true) — when true, enforce sets password aging on accounts with no expiry
   - user_audit_fix_service_shells (bool, default false) — when true, enforce changes service account interactive shells to nologin
   - user_audit_skip_users (list of str, default [root, halt, sync, shutdown]) — accounts never touched by enforce tasks
