## Objective

Create the Jinja2 template for chrony.conf that renders a hardened configuration from role variables. The template must never include `monitor` or `cmdallow` directives (CVE-2013-5211 mitigation).

Purpose: Deploy a secure, configurable chrony.conf to the OS-appropriate path.
Output: templates/chrony.conf.j2 ready for the enforce.yml template task.

## Tasks

<task type="auto">
  <name>Task 1: Write templates/chrony.conf.j2</name>
  <read_first>
    - roles/ntp_hardening/defaults/main.yml
    - .planning/phases/01-ntp-hardening/01-RESEARCH.md
  </read_first>
  <files>roles/ntp_hardening/templates/chrony.conf.j2</files>
  <action>
Write `templates/chrony.conf.j2` with this exact content:

```jinja2
# Managed by Ansible — malpanez.security.ntp_hardening
# Do not edit manually.

{% for server in ntp_hardening_servers %}
server {{ server }}
{% endfor %}

{% if ntp_hardening_allow %}
{% for net in ntp_hardening_allow %}
allow {{ net }}
{% endfor %}
{% endif %}
{% if ntp_hardening_deny_all %}
deny all
{% endif %}

makestep {{ ntp_hardening_makestep }}
{% if ntp_hardening_rtcsync %}
rtcsync
{% endif %}
maxdistance {{ ntp_hardening_maxdistance }}
minsources {{ ntp_hardening_minsources }}
leapsectz {{ ntp_hardening_leapsectz }}

logdir {{ ntp_hardening_logdir }}
log tracking measurements statistics
logchange {{ ntp_hardening_logchange }}

# CVE-2013-5211: monitor directive intentionally absent.
# Do NOT add monitor or cmdallow directives.
```

CRITICAL SECURITY CONSTRAINT: The template MUST NOT contain the words `monitor` or `cmdallow` anywhere except in the comment block at the bottom. The words must only appear preceded by `#` (comment character). Verify by searching for uncommented occurrences.

CRITICAL: The comment is important for auditability — it documents WHY the directive is absent. Keep the comment exactly as shown.
  </action>
