# Runbook Debian 13: SSH keys only + sudo with TOTP (YubiKey OATH)

## Initial variables

```
ADMIN_USER="<my_admin_user>"
MFA_GROUP="mfa-required"
MFA_SECRET_BASE="/var/lib/pam-google-authenticator"
SSHD_DROPIN="/etc/ssh/sshd_config.d/20-auth-hardening.conf"
```

## Diagnosis (before touching anything)

```
cat /etc/os-release
sshd -V
grep -E '^[[:space:]]*Include[[:space:]]+.*/sshd_config.d/\\*\\.conf' /etc/ssh/sshd_config
sshd -T | egrep -i 'passwordauthentication|kbdinteractiveauthentication|challengeresponseauthentication|authenticationmethods|pubkeyacceptedalgorithms'
ls -la /etc/ssh/sshd_config.d/
grep -n '' /etc/pam.d/sudo | sed -n '1,120p'
pam-auth-update --list
```

Keep an SSH session open throughout the entire change.

## Design (options)

1) Clean Debian 13 (recommended): drop-in in `sshd_config.d` + sudo with substack `mfa-totp` + TOTP only for group `mfa-required`. Does not touch `common-*`.
2) Global via pam-auth-update: enables MFA in `common-auth` (affects login/sshd). Only if you want MFA system-wide.
3) Legacy: edit `/etc/pam.d/sudo` directly without substack (not recommended).

Option 1 is chosen for isolation and lower risk of lockout.

## Manual implementation (step by step)

### 1) Required packages

```
apt-get update
apt-get install -y openssh-server libpam-google-authenticator yubikey-manager pamtester
```

### 2) SSH (public keys only, FIDO2 preference)

Create drop-in at `SSHD_DROPIN`:

```
cat > "$SSHD_DROPIN" <<'EOF'
# Managed by policy
PasswordAuthentication no
ChallengeResponseAuthentication no
KbdInteractiveAuthentication no
AuthenticationMethods publickey
# Optional (gradual): restrict to SK keys with Match/User/Group
# PubkeyAcceptedAlgorithms sk-ssh-ed25519@openssh.com,sk-ecdsa-sha2-nistp256@openssh.com
UsePAM yes
PermitRootLogin no
EOF
```

Validate and reload:

```
sshd -t
systemctl reload ssh
```

### 3) PAM sudo with TOTP (substack)

Create MFA group and directories:

```
groupadd -f "$MFA_GROUP"
usermod -a -G "$MFA_GROUP" "$ADMIN_USER"
install -d -m 0700 -o root -g root "$MFA_SECRET_BASE"
install -d -m 0700 -o root -g root "$MFA_SECRET_BASE/$ADMIN_USER"
```

Create substack `/etc/pam.d/mfa-totp`:

```
cat > /etc/pam.d/mfa-totp <<'EOF'
#%PAM-1.0
auth [success=1 default=ignore] pam_succeed_if.so user notingroup mfa-required
auth required pam_google_authenticator.so secret=/var/lib/pam-google-authenticator/%u/.google_authenticator user=root allowed_perm=0400
EOF
```

Include substack in sudo (before `common-auth`):

```
cp -a /etc/pam.d/sudo /etc/pam.d/sudo.backup.$(date +%s)
awk 'NR==1{print; print "auth    include    mfa-totp"; next}1' /etc/pam.d/sudo > /etc/pam.d/sudo.new
install -m 0644 /etc/pam.d/sudo.new /etc/pam.d/sudo
rm -f /etc/pam.d/sudo.new
```

### 4) Generate TOTP secret (server-side)

Avoid shell history:

```
set +o history
google-authenticator -t -f -r 3 -R 30 -W -s "$MFA_SECRET_BASE/$ADMIN_USER/.google_authenticator"
chown root:root "$MFA_SECRET_BASE/$ADMIN_USER/.google_authenticator"
chmod 0400 "$MFA_SECRET_BASE/$ADMIN_USER/.google_authenticator"
set -o history
```

### 5) Provision YubiKey OATH/TOTP

```
ykman oath accounts add --touch "sudo:$HOSTNAME:$ADMIN_USER"
ykman oath accounts list
ykman oath accounts code "sudo:$HOSTNAME:$ADMIN_USER"
ykman oath access change
```

When `ykman` asks for the secret, paste the same secret generated by `google-authenticator`. Do not write it on the command line.

## Ansible implementation (recommended)

```
ADMIN_USER="$ADMIN_USER" MFA_GROUP="$MFA_GROUP" ansible-playbook playbooks/debian13-auth-hardening.yml -i <your_inventory>
```

To enable faillock via pam-auth-update (optional):

```
ansible-playbook playbooks/debian13-auth-hardening.yml -i <your_inventory> \
  -e pam_mfa_enable_faillock=true
```

faillock verification:

```
grep -n "pam_faillock.so" /etc/pam.d/common-auth
pam-auth-update --list | grep faillock
```

For gradual SK-only by human group:

```
ansible-playbook playbooks/debian13-auth-hardening.yml -i <your_inventory> \
  -e sshd_hardening_human_pubkey_accepted_algorithms='["sk-ssh-ed25519@openssh.com","sk-ecdsa-sha2-nistp256@openssh.com"]'
```

## Verification

Before:

```
sshd -T | egrep -i 'passwordauthentication|kbdinteractiveauthentication|authenticationmethods|pubkeyacceptedalgorithms'
grep -n "mfa-totp" /etc/pam.d/sudo
```

After (from another session):

```
ssh -o PreferredAuthentications=publickey -o PubkeyAcceptedAlgorithms=sk-ssh-ed25519@openssh.com <host>
sudo -k
sudo -v   # Should ask for local password and TOTP code
```

## Rollback

```
cp -a /etc/pam.d/sudo.backup.* /etc/pam.d/sudo
rm -f /etc/pam.d/mfa-totp
rm -f "$SSHD_DROPIN"
sshd -t
systemctl reload ssh
```

## Gotchas

- If `/etc/ssh/sshd_config` does not include `sshd_config.d/*.conf`, the drop-in does not apply. Enable Include first.
- `pam-auth-update` can alter `common-auth`; do not use it for MFA if you only want it for sudo.
- `pam_google_authenticator` requires a secret on the server; do not use `nullok` to avoid bypass.
- Always validate with `sshd -t` before reloading; keep an active session open.
